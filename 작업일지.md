## 작업 순서

### 1단계 — 기존 파일 파악

기존 `quiz-app/` 구조 확인:

```
quiz-app/
├── index.html      ← HTML 뼈대 (외부 CSS·JS 참조)
├── quiz.js         ← 바닐라 JS 로직 (~1,000줄)
├── style.css       ← 공용 스타일 (~1,500줄)
└── data/
    └── questions.json  ← 샘플 문항 데이터
```

**핵심 확인 사항**
- 기존 구조는 `index.html` + `quiz.js` + `style.css` 3파일 분리 방식
- Claude API 직접 호출 (`fetch` + `anthropic-dangerous-direct-browser-calls: true`)
- `file://` 환경에서 CORS 오류 발생 가능성 있음
- `teacher-dashboard.html` 별도 존재 (이번 작업에서 불필요 판단, 제거)

### 2단계 — 단일 HTML 파일로 통합 재구성

**목표:** `quiz-app/index.html` 하나로 완성형 앱 구현

**변경 내용**

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 파일 구조 | `index.html` + `quiz.js` + `style.css` | `index.html` 단일 파일 |
| JS 프레임워크 | 바닐라 JS (DOM 직접 조작) | React 18 + Babel standalone CDN |
| 상태 관리 | `appState` 전역 객체 | React `useState` 훅 |
| 스타일 | 외부 `style.css` | `<style>` 인라인 |
| 스크립트 | 외부 `quiz.js` | `<script type="text/babel">` 인라인 |

**React 컴포넌트 구조**

```
App (메인 상태 관리)
├── AILoadingOverlay      ← Claude API 호출 중 오버레이
├── UnansweredModal       ← 미응답 경고 모달
├── Header                ← 단계 뱃지 포함
└── 화면 라우터 (screen 상태로 전환)
    ├── ApiKeyScreen      ← 화면 1: API 키 입력
    ├── GradeScreen       ← 화면 2: 급수 선택 (1~6급)
    ├── GoalScreen        ← 화면 3: 학습 목표 + 평가 영역
    ├── RubricScreen      ← 화면 4: 루브릭 검토·수정
    ├── QuestionsScreen   ← 화면 5: 생성 문항 미리보기
    ├── QuizScreen        ← 화면 6: 문항 풀기
    │   ├── GrammarMC         (문법 객관식)
    │   ├── GrammarTransform  (문법 변형형)
    │   ├── ReadingMC         (읽기 객관식)
    │   ├── ListeningMC       (듣기 객관식)
    │   └── Performance       (수행형)
    └── ResultScreen      ← 화면 7: 결과 + 루브릭 피드백
```

**7단계 플로우**

```
[1] API 키 입력 (localStorage 저장/복원)
    ↓
[2] 급수 선택 (1~6급)
    ↓
[3] 학습 목표 + 평가 영역 입력
    ↓ Claude API → 루브릭 자동 생성
[4] 루브릭 검토 / 교사 수정 가능
    ↓ Claude API → 문항 자동 생성
[5] 생성 문항 검토 (다시 생성 가능)
    ↓
[6] 학습자 퀴즈 화면 (5종 문항 유형)
    ↓
[7] 결과 화면 (자동채점 + 수행형 루브릭 피드백)
```

**Claude API 호출 방식**

```js
const res = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: {
    'x-api-key': apiKey,
    'anthropic-version': '2023-06-01',
    'content-type': 'application/json',
    'anthropic-dangerous-direct-browser-calls': 'true'   // 브라우저 직접 호출 허용
  },
  body: JSON.stringify({
    model: 'claude-sonnet-4-6',
    max_tokens: 4096,
    system: systemPrompt,
    messages: [{ role: 'user', content: userPrompt }]
  })
});
```

### 4단계 — CDN 오류 수정

**오류:** `Cannot use import statement outside a module`

**원인:** `unpkg.com`의 `.production.min.js` 파일이 일부 환경에서 ES module로 해석됨

**수정:** CDN을 `cdnjs.cloudflare.com`으로 전환 + `.development.js` → `production.min.js` 고정 버전 사용

```html
<!-- 변경 전 (unpkg) -->
<script src="https://unpkg.com/react@18/umd/react.development.js"></script>
<script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>

<!-- 변경 후 (cdnjs, 고정 버전) -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/react/18.3.1/umd/react.production.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/react-dom/18.3.1/umd/react-dom.production.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/babel-standalone/7.23.5/babel.min.js"></script>
```

### 5단계 — 로컬 개발용 프록시 서버 생성

**파일:** `quiz-app/proxy.js`

**목적:** `file://` 환경 CORS 우회 없이 `http://localhost:3000`에서 안전하게 실행

**특징**
- Node.js 내장 모듈만 사용 (`http`, `https`, `fs`, `path`) → 외부 패키지 설치 불필요
- `POST /api/claude` → `https://api.anthropic.com/v1/messages` 프록시
- `GET /` → `index.html` 정적 서빙
- CORS 헤더 전체 허용

**로컬 실행 방법**
```bash
node quiz-app/proxy.js
# → http://localhost:3000 접속
```

> **주의:** `proxy.js` 사용 시 `callClaude`의 fetch URL을 `/api/claude`로, body에 `apiKey` 포함 필요.  
> GitHub Pages 배포 시에는 직접 API 호출 방식으로 복원해야 함.

### 6단계 — GitHub Pages 배포

**배포 방식:** GitHub Pages (정적 호스팅)  
**이유:** proxy.js는 로컬 전용 → 브라우저 직접 API 호출 방식으로 복원 후 배포

**callClaude 복원**

```js
// GitHub Pages 배포용 (직접 API 호출)
fetch('https://api.anthropic.com/v1/messages', {
  headers: {
    'x-api-key': apiKey,
    'anthropic-dangerous-direct-browser-calls': 'true',
    ...
  }
})
```

**배포 절차**

```bash
# quiz-app/ 폴더에서 git 초기화
cd quiz-app/
git init
git checkout -b main

# 파일 스테이징 (index.html, proxy.js, .gitignore)
git add index.html proxy.js .gitignore
git commit -m "한국어 QUIZ 앱 초기 배포"

# 원격 레포 연결 후 push
git remote add origin https://github.com/gmountain01/learning_korean.git
git push -u origin main

# GitHub Pages 활성화 (gh CLI)
gh api repos/gmountain01/learning_korean/pages \
  -X POST \
  -f source[branch]=main \
  -f source[path]=/
```

## 참고 사항

- **API 키 보안:** 브라우저 localStorage에만 저장, 외부 서버로 전송 안 됨
